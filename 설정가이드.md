# 갠홈 설정 가이드 (코드 몰라도 따라할 수 있어요)

이 갠홈은 파일 5개로 이루어져 있어요.
- `index.html` — 페이지 뼈대
- `style.css` — 디자인(방안지 배경, 색, 애니메이션)
- `app.js` — 실제 동작(위젯 추가/편집/저장)
- `firebase-config.js` — 데이터 저장소 연결 정보 (⚠️ 꼭 직접 채워야 함)
- `설정가이드.md` — 지금 보고 있는 이 문서

전체 순서는 **① Firebase 만들기 → ② config 값 붙여넣기 → ③ GitHub Pages로 올리기** 입니다.

---

## ① Firebase 프로젝트 만들기 (무료, 5분)

갠홈에서 위젯 내용·색상·비밀번호를 저장하려면 데이터 저장소가 필요해요. GitHub Pages 자체는 저장 기능이 없어서, 구글의 무료 서비스인 **Firebase**를 붙여씁니다. (방문자 수가 아주 많지 않다면 완전히 무료예요.)

1. https://console.firebase.google.com 접속 → 구글 계정으로 로그인
2. **프로젝트 추가** 클릭 → 이름 입력(예: `myganhome`) → 애널리틱스는 꺼도 됨 → 프로젝트 만들기
3. 왼쪽 메뉴 **빌드 > Firestore Database** 클릭 → **데이터베이스 만들기**
   - 위치는 `asia-northeast3(서울)` 선택
   - 보안 규칙은 우선 **테스트 모드**로 시작 (아래 ④에서 규칙을 다시 손볼 거예요)
4. 왼쪽 메뉴 **빌드 > Firestore Database > 규칙** 탭에서 아래 내용으로 바꾸고 **게시**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read: if true;
      allow write: if true;
    }
  }
}
```

> ℹ️ 이 규칙은 "누구나 읽고 쓸 수 있음"이에요. 갠홈의 비밀번호 잠금은 **화면(UI)에서만** 막아주는 거라, 아주 작정하고 개발자 도구를 열어 직접 접근하는 사람까지 막지는 못해요. 다만 그 정도로 시도할 사람은 거의 없을 테니, 개인 갠홈 용도로는 충분해요. 더 강한 보안이 필요해지면 나중에 "Firebase Authentication"을 추가하는 방법이 있으니, 그때 다시 요청해주세요.

## ①-1 사진·음악은 어떻게 넣나요? (무료, 카드 등록 없이)

⚠️ Firebase Storage(파일 직접 업로드 기능)는 2026년 2월부터 카드 등록이 필요한 유료 요금제(Blaze)에서만 쓸 수 있게 바뀌었어요. 그래서 이 갠홈은 **직접 업로드 대신, 무료 사이트에 올린 뒤 그 주소(링크)를 붙여넣는 방식**을 써요. 카드 등록도, 돈도 전혀 안 들어요.

**사진**
1. https://imgbb.com 또는 https://postimages.org 접속 (가입 없이도 가능)
2. 사진 업로드 → 나오는 여러 링크 중 **"직접 링크(Direct link)"** 를 복사
3. 갠홈의 갤러리/배경사진 등 "이미지 URL" 칸에 붙여넣기

**음악**
1. 구글 드라이브에 음악 파일 업로드 → 파일 우클릭 **공유 > 일반 액세스: 링크가 있는 모든 사용자** → 링크 복사
2. 복사한 링크에서 `.../file/d/여기가파일ID/view?usp=sharing` 형태 중 **파일ID** 부분만 떼어내서, 아래 형태로 바꿔주세요:
   `https://drive.google.com/uc?export=download&id=파일ID`
3. 이 주소를 음악 플레이어의 "오디오 파일 URL"에 붙여넣기

**자료 백업 (문서, PDF 등)**
- 구글 드라이브 공유 링크를 그대로 백업 카드의 "링크"에 붙여넣으면 돼요 (직접 다운로드 형태로 안 바꿔도 됩니다).

## ② 내 프로젝트 설정 값 붙여넣기

1. Firebase 콘솔 왼쪽 위 **⚙️ 프로젝트 설정** 클릭
2. 아래로 스크롤 → **내 앱** → 웹 아이콘(`</>`) 클릭 → 앱 닉네임 아무거나 입력 → 앱 등록
3. 화면에 나오는 `firebaseConfig = { ... }` 값을 통째로 복사
4. 제가 드린 `firebase-config.js` 파일을 메모장으로 열고, `여기에_...` 부분을 방금 복사한 실제 값으로 교체 후 저장

## ③ GitHub Pages로 올리기

1. https://github.com 가입/로그인 → 오른쪽 위 **+** → **New repository**
2. 저장소 이름 입력(예: `our-ganhome`) → Public 선택 → Create repository
3. **Add file > Upload files** 클릭 → 위 5개 파일(index.html, style.css, app.js, firebase-config.js, 이 가이드는 선택)을 전부 끌어다 놓기 → Commit changes
4. 저장소 상단 **Settings > Pages** 이동
5. **Build and deployment > Branch**에서 `main` / `/(root)` 선택 → Save
6. 1~2분 뒤 같은 페이지에 `https://아이디.github.io/our-ganhome/` 주소가 생겨요. 그게 갠홈 주소예요!

## 사용법

- **처음 잠금 해제**: 오른쪽 위 "잠금 해제" 클릭 → 처음엔 비밀번호를 새로 정하는 창이 떠요. 여기서 정한 비밀번호를 앞으로 함께 쓸 사람에게만 알려주세요.
- **배너**: 이제 위젯이 아니라 페이지 맨 위에 항상 고정돼요. 잠금 해제 상태면 배너 오른쪽 아래 "배너 사진/문구 편집" 버튼으로 사진 URL과 부제목을 바꿀 수 있어요.
- **위젯 추가**: 잠금 해제 상태에서 화면 아래 "위젯 추가" 줄의 버튼 클릭 (배너와 커미션은 목록에서 빠졌어요 — 커미션이 필요하면 "갤러리" 위젯을 하나 더 추가해서 이미지만 올리는 용도로 쓰면 돼요. 갤러리는 위젯 하나 안에서 크고 작은 그림을 섞어 보여주는 벤토 레이아웃이에요)
- **제목 바꾸기**: 잠금 해제 상태에서 위젯 제목을 그냥 클릭해서 바로 수정, 다른 곳 클릭하면 저장
- **위젯 개별 색/배경사진/크기**: 위젯 오른쪽 위 ⚙️ 클릭
- **전체 스타일 한 번에 바꾸기**: 맨 위 "🎨 전체 스타일" 버튼 → 포인트 컬러·배경색·카드색·폰트를 한 번에 모든 위젯에 적용. "위젯별로 따로 지정한 색상 전부 초기화"를 체크하면 개별 설정을 지우고 전체 스타일이 확실히 보이게 해줘요.
- **사이트 이름**: 맨 위 왼쪽 글자도 잠금 해제 상태면 바로 수정 가능
- **음악 플레이어**: 곡 추가 시 mp3 직링크 대신 유튜브 링크(`youtube.com/watch?v=...`, `youtu.be/...`)를 그대로 붙여넣어도 돼요. 곡을 누르면 화면 안에서 바로 재생돼요.
- **갤러리·백업 카드·위젯 배경사진**: 전부 "URL(링크) 붙여넣기" 방식이에요. 위 "①-1" 안내대로 무료 사이트에 올린 링크를 붙여넣으면 돼요.

## 문제 해결

- **잠금 해제를 눌러도 아무 반응이 없어요 / 위젯이 하나도 안 보여요**: 화면 위쪽에 빨간 안내 배너가 뜬다면 `firebase-config.js`에 아직 실제 값을 안 넣은 상태예요. ②단계를 다시 확인해주세요. 안내 배너도 없다면, Firestore 규칙(①단계 4번)이 게시됐는지, 브라우저 개발자도구(F12) > Console 탭에 빨간 에러가 있는지 확인해주세요.
- **외부 링크 임베드**: 사이트에 따라 보안 설정 때문에 화면 삽입이 막혀 있을 수 있어요(예: 대부분의 SNS). 그런 경우엔 "새 창에서 열기" 링크만 동작해요. 잘 임베드되는 대표적인 예시는 유튜브 영상 주소를 `/watch?v=` 대신 `/embed/`로 바꾼 링크, 구글 지도/캘린더 공유 embed 주소 등이에요.
- **비밀번호를 바꾸고 싶을 때**: 현재 버전은 최초 설정한 비밀번호를 그대로 씁니다. 바꾸고 싶으면 Firebase 콘솔 > Firestore Database에서 `meta` 컬렉션의 `lock` 문서를 통째로 삭제한 뒤, 갠홈에서 다시 "잠금 해제"를 누르면 새 비밀번호를 정하는 창이 다시 떠요.

## 알아두면 좋은 점

- 데이터는 전부 Firebase에 저장되어서, 어느 기기·브라우저로 들어와도 모두 같은 내용을 봐요.
- 무료 사용량을 넘지 않는 한 비용은 들지 않아요(친구들끼리 보는 개인 갠홈 규모라면 거의 걱정 없는 수준).
- 나중에 기능을 더 추가하고 싶으면(실제 파일 업로드, 배경음악 자동재생, 위젯 순서 드래그 정렬 등) 언제든 이어서 요청해주세요.
