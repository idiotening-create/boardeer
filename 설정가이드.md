# 노은 갠홈 설정 가이드 (코드 몰라도 따라할 수 있어요)

이번 버전은 위젯을 자유롭게 추가/배치하던 이전 방식과 다르게, **정해진 8개 영역(이미지 슬라이드·음악·디데이·방명록·캘린더·갤러리·세션카드·체크보드)이 항상 같은 구성으로 고정되어** 보여요. 내용을 채우고 꾸미는 것만 화면에서 바로 할 수 있어요.

이 갠홈은 파일/폴더 6개로 이루어져 있어요.
- `index.html` — 페이지 뼈대 (고정 레이아웃 구조)
- `style.css` — 디자인(반투명 유리 카드, 색, 폰트)
- `app.js` — 실제 동작(각 영역 편집/저장)
- `firebase-config.js` — 데이터 저장소 연결 정보 (⚠️ 꼭 직접 채워야 함)
- `fonts/ZenSerif.ttf` — 기본 글자체 파일 (제니가 만든 무료 폰트 ZEN SERIF). **반드시 `fonts`라는 폴더를 만들고 그 안에 이 파일을 넣어야** 사이트에서 폰트가 정상적으로 보여요.
- `설정가이드.md` — 지금 보고 있는 이 문서

전체 순서는 **① Firebase 만들기 → ② config 값 붙여넣기 → ③ GitHub Pages로 올리기** 입니다.

## ① Firebase 프로젝트 만들기 (무료, 5분)

갠홈에서 내용·색상·비밀번호를 저장하려면 데이터 저장소가 필요해요. GitHub Pages 자체는 저장 기능이 없어서, 구글의 무료 서비스인 **Firebase**를 붙여씁니다. (방문자 수가 아주 많지 않다면 완전히 무료예요.)

1. https://console.firebase.google.com 접속 → 구글 계정으로 로그인
2. **프로젝트 추가** 클릭 → 이름 입력(예: `noeun-home`) → 애널리틱스는 꺼도 됨 → 프로젝트 만들기
3. 왼쪽 메뉴 **빌드 > Firestore Database** 클릭 → **데이터베이스 만들기**
   - 위치는 `asia-northeast3(서울)` 선택
   - 보안 규칙은 우선 **테스트 모드**로 시작 (아래 4번에서 규칙을 다시 손볼 거예요)
4. 왼쪽 메뉴 **빌드 > Firestore Database > 규칙** 탭에서 아래 내용으로 바꾸고 **게시**:

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read: if true;
      allow write: if true;
    }
  }
}
```

> ℹ️ 이 규칙은 "누구나 읽고 쓸 수 있음"이에요. 갠홈의 비밀번호 잠금은 **화면(UI)에서만** 편집 버튼들을 숨기는 거라, 아주 작정하고 개발자 도구를 열어 직접 접근하는 사람까지 막지는 못해요. (방명록은 애초에 누구나 남길 수 있게 설계되어 있어서 이 규칙과 잘 맞아요.) 더 강한 보안이 필요해지면 나중에 "Firebase Authentication"을 추가하는 방법이 있으니, 그때 다시 요청해주세요.

## ①-1 사진·음악·PDF는 어떻게 넣나요?

⚠️ Firebase Storage(파일 직접 업로드 기능)는 유료 요금제에서만 쓸 수 있어서, 이 갠홈은 **사진을 화면에서 바로 선택하면 자동으로 압축해서 Firestore에 저장**하는 방식을 써요. 카드 등록도, 돈도 전혀 안 들어요. (배너·이미지 슬라이드·갤러리 전부 이 방식)

- 사진이 너무 크거나 여러 장을 한꺼번에 많이 올리면 저장이 안 될 수 있어요. 그럴 땐 아래처럼 외부 링크 방식을 대신 쓰면 돼요.
  1. https://imgbb.com 또는 https://postimages.org 접속 (가입 없이도 가능)
  2. 사진 업로드 → **"직접 링크(Direct link)"** 복사 → 갠홈의 "이미지 URL" 칸에 붙여넣기

**음악**
1. 유튜브 링크는 그냥 그대로 붙여넣으면 돼요 (`youtube.com/watch?v=...` 또는 `youtu.be/...`).
2. mp3 등 직접 파일은 구글드라이브에 올린 뒤 파일 우클릭 **공유 > 일반 액세스: 링크가 있는 모든 사용자** → 복사한 링크에서 `.../file/d/여기가파일ID/view` 중 **파일ID**만 떼어내 `https://drive.google.com/uc?export=download&id=파일ID` 형태로 바꿔 붙여넣어주세요.

**문서 정리 위젯 파일 / TRPG 세션카드의 PDF**
- 약 8MB까지는 파이어 스토리지(유료) 없이, 화면에서 파일을 바로 선택하면 그대로 저장돼요.
  - 약 650KB보다 작으면 카드 안에 바로 저장되고, 그보다 크면(최대 8MB) 뒤에서 자동으로 여러 개의 조각으로 나눠 Firestore에 저장한 뒤 열 때 다시 합쳐서 보여줘요. 화면에서 조작하는 방법은 똑같고, 용량이 크면 저장/열기에 몇 초 더 걸릴 수 있어요.
- 8MB보다 큰 파일은 구글드라이브에 올리고 공유 링크를 "링크로 연결"에 붙여넣어주세요. (드라이브 공유 링크는 대부분 클릭하면 PDF 미리보기가 바로 열려요)

## ② 내 프로젝트 설정 값 붙여넣기

1. Firebase 콘솔 왼쪽 위 **⚙️ 프로젝트 설정** 클릭
2. 아래로 스크롤 → **내 앱** → 웹 아이콘(`</>`) 클릭 → 앱 닉네임 아무거나 입력 → 앱 등록
3. 화면에 나오는 `firebaseConfig = { ... }` 값을 통째로 복사
4. 제가 드린 `firebase-config.js` 파일을 메모장으로 열고, 값을 방금 복사한 실제 값으로 교체 후 저장 (지금 드린 파일은 이미 채워져 있는 상태라, 새 프로젝트를 만들지 않았다면 그대로 두셔도 돼요)

## ③ GitHub Pages로 올리기 / 주소(도메인) 만들기·바꾸기

1. https://github.com 가입/로그인 → 오른쪽 위 **+** → **New repository**
2. 저장소 이름 입력(예: `noeun`) → Public 선택 → Create repository
3. **Add file > Upload files** 클릭 → `index.html`, `style.css`, `app.js`, `firebase-config.js`, 이 가이드(선택)를 전부 끌어다 놓기 → 이어서 컴퓨터의 `fonts` 폴더째로(폴더 안에 `ZenSerif.ttf`가 들어있는 상태로) 같이 끌어다 놓으면 저장소 안에 `fonts/ZenSerif.ttf` 경로가 그대로 만들어져요 → Commit changes
   > 폴더째 드래그가 안 되는 브라우저라면, 저장소 페이지에서 먼저 파일명 입력칸에 `fonts/ZenSerif.ttf`라고 쳐서 폴더를 만든 뒤 그 안에 폰트 파일만 업로드해도 돼요.
4. 저장소 상단 **Settings > Pages** 이동
5. **Build and deployment > Branch**에서 `main` / `/(root)` 선택 → Save
6. 1~2분 뒤 같은 페이지에 `https://아이디.github.io/noeun/` 주소가 생겨요. 그게 갠홈 주소예요!

### 나중에 주소를 바꾸고 싶을 때

- **가장 쉬운 방법 (완전 무료)**: 저장소 페이지 상단 **Settings > 맨 위 저장소 이름(Repository name)** 을 원하는 이름으로 바꾸고 저장 → 주소가 `https://아이디.github.io/새이름/` 으로 자동으로 바뀌어요. 기존 주소로 들어와도 한동안은 새 주소로 자동 연결돼요.
- **완전히 새로 만들고 싶다면**: ①~③ 과정을 새 저장소 이름으로 한 번 더 하면 돼요 (Firebase 프로젝트는 그대로 재사용해도, 새로 만들어도 상관없어요).
- **내가 가진 개인 도메인(예: noeun.com)을 쓰고 싶다면**: 도메인은 가비아·후이즈·Namecheap 같은 곳에서 별도로 구입해야 해요(보통 연 1만원대~, 이건 GitHub/Firebase와 무관한 별도 비용이에요). 도메인을 구입한 뒤 **Settings > Pages > Custom domain**에 그 도메인을 입력하면 돼요. 이 부분은 도메인을 실제로 구입하신 뒤에 알려주시면, 정확한 DNS 설정 값까지 같이 잡아드릴게요.

## 사용법 (화면에서 바로 편집하기)

- **처음 잠금 해제**: 오른쪽 위 "잠금 해제" 클릭 → 처음엔 비밀번호를 새로 정하는 창이 떠요. 이 비밀번호를 아는 사람만 내용을 수정할 수 있어요.
- **사이트 이름 / 배너 부제목**: 잠금 해제 상태에서 텍스트를 클릭해서 바로 수정, 다른 곳 클릭하면 저장돼요.
- **배너 이미지**: 배너 오른쪽 아래 "배너 이미지 변경" 버튼 → 기기의 사진을 바로 선택하면 코드나 다른 사이트 없이 갠홈에서 바로 배너 사진이 올라가요.
- **테마 편집**: 상단 "🎨 테마 편집" 버튼 → 포인트 컬러·배경색·카드색·글자색과 제목/본문 폰트를 사이트 전체에 한 번에 적용. 아래쪽 "커스텀 폰트 파일" 항목에서 다른 `.ttf`/`.otf` 파일을 올리면 사이트 전체 글자체가 그 폰트 하나로 통일돼요 (500KB 초과 시 `fonts` 폴더에 올리고 파일명만 입력).
- **이미지 위젯 (가로 슬라이드)**: 몇 초마다 자동으로 다음 사진으로 넘어가고, 마우스를 올리면 잠깐 멈춰요. 잠금 해제 상태면 "+ 사진 추가"로 여러 장을 한 번에 올릴 수 있고, 사진 위 ✕ 버튼으로 지금 보이는 사진을 지울 수 있어요.
- **음악 플레이어**: "+ 곡 추가"로 곡을 등록하고, 곡 이름을 누르면 재생돼요. 유튜브 링크는 화면 안에서 바로 재생.
- **디데이**: 제목("D-Day")도 클릭해서 바꿀 수 있어요. "+ 디데이 추가"로 이름과 날짜를 등록하면 자동으로 D-값이 계산돼요. 항목이 많아지면 안에서 스크롤로 더 볼 수 있어요.
- **방명록**: 제목("방명록")도 잠금 해제 상태면 수정 가능해요. **글을 남기는 건 잠금 상태와 상관없이 누구나 할 수 있어요** — 이름과 메시지를 적고 "남기기"만 누르면 돼요. 삭제(✕)는 편집모드(잠금 해제 상태)에서만 보여요. 목록이 길어지면 안에서 스크롤로 더 볼 수 있어요.
- **캘린더**: 날짜를 누르면 그날의 일정을 볼 수 있고, 잠금 해제 상태면 그 자리에서 바로 일정을 적거나 지울 수 있어요.
- **갤러리**: 사진 크기가 자연스럽게 섞여 보이는 핀터레스트 스타일이에요. 잠금 해제 상태면 우측 하단 동그란 ＋ 버튼으로 사진을 추가하고, 사진을 누르면 크게 보면서 삭제할 수 있어요.
- **문서 정리**: "+ 문서 추가"로 아이콘·제목·설명과 문서(링크 또는 파일)를 등록해요. **각 항목 오른쪽 ✎ 버튼으로 언제든 내용을 수정**할 수 있어요. "⚙ 옵션 관리" 버튼으로 부제처럼 붙일 옵션 목록(예: 설정집/규칙/자료 등 원하는 이름)을 직접 만들고 지울 수 있고, 문서를 추가/수정할 때 그 목록 중 하나를 골라 제목 아래 부제로 붙일 수 있어요.
- **세션카드 (TRPG)**: "+ 자료 추가"로 썸네일 사진·제목·PDF(또는 링크)를 등록하면 카드가 2열로 쌓여요. **썸네일 사진은 16:9 가로 이미지를 기준으로 카드가 표시돼요** (세로 사진을 올리면 위아래가 잘려 보일 수 있어요). 카드를 누르면 연결된 자료가 새 창으로 열리고, 카드 좌측 위 ✎ 버튼으로 제목·사진·PDF를 수정할 수 있어요. 카드가 많아지면 안에서 스크롤돼요.
- **체크보드**: 잠금 해제 상태에서 아래 입력칸으로 항목을 추가하고 체크박스를 누르면, **체크된 항목은 자동으로 목록 맨 아래(점선 구분선 아래)로 내려가요.** 항목이 많아지면 안에서 스크롤돼요.

## 문제 해결

- **잠금 해제를 눌러도 아무 반응이 없어요 / 화면이 비어 있어요**: 화면 위쪽에 빨간 안내 배너가 뜬다면 `firebase-config.js`에 아직 실제 값을 안 넣은 상태예요. ②단계를 다시 확인해주세요. 안내 배너도 없다면, Firestore 규칙(①단계 4번)이 게시됐는지, 브라우저 개발자도구(F12) > Console 탭에 빨간 에러가 있는지 확인해주세요.
- **폰트가 기본 서체로 보여요**: `fonts/ZenSerif.ttf` 경로가 저장소에 정확히 있는지 확인해주세요. 대소문자까지 정확히 일치해야 해요.
- **PDF/사진이 저장이 안 돼요**: 파일 용량이 8MB(사진은 별도 안내 용량)보다 큰 경우예요. 안내된 대로 링크(구글드라이브 등) 방식을 대신 이용해주세요.
- **문서/PDF를 열었는데 "불러오지 못했어요"가 떠요**: 8MB 근처의 큰 파일은 여러 조각으로 나눠 저장돼 있어서 불러오는 데 시간이 조금 걸려요. 인터넷이 불안정하면 실패할 수 있으니 다시 눌러봐주세요.
- **비밀번호를 바꾸고 싶을 때**: Firebase 콘솔 > Firestore Database에서 `meta` 컬렉션의 `lock` 문서를 통째로 삭제한 뒤, 갠홈에서 다시 "잠금 해제"를 누르면 새 비밀번호를 정하는 창이 다시 떠요.

## 알아두면 좋은 점

- 이번 버전은 위젯을 마음대로 추가/이동하던 방식에서, **정해진 8개 영역이 항상 같은 자리에 있는 고정 레이아웃**으로 완전히 새로 만들었어요. 그만큼 위젯 설정(⚙)·드래그 순서 변경·위젯 추가바 같은 예전 기능은 없어졌어요.
- 데이터는 Firestore의 `content` 컬렉션 안에 영역별로(`images`, `music`, `dday`, `guestbook`, `calendar`, `gallery`, `documents`, `docOptions`, `sessions`, `checklist`) 각각 저장돼요. 어느 기기·브라우저로 들어와도 모두 같은 내용을 봐요.
- 문서 위젯이나 세션카드에 650KB보다 큰 파일을 올리면, 그 파일은 `fileChunks`라는 별도 컬렉션에 여러 조각으로 나뉘어 저장돼요. 카드를 삭제하면 이 조각들도 자동으로 함께 삭제돼요.
- 무료 사용량을 넘지 않는 한 비용은 들지 않아요.
